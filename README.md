# 中文文章智能分析助手

一个基于 **NLP** 的中文文本实体识别、可视化分析与智能对话系统。支持文本、图片 OCR、URL 抓取、文件上传、批量处理等多种输入方式，适用于各类中文文章分析场景。

## 功能特点

- **双引擎 NER**：支持 `jieba` 离线规则引擎和 `GLM` 在线大模型引擎，可自由切换。
- **智能文章总结**：支持抽取式摘要和 GLM 大模型生成式摘要。
- **丰富的实体类型**：人名、组织机构、国家/地区/城市、地点、时间、活动/事件、专业/课程、职务/职称、产品/平台、技术术语、法律法规/政策、金额、百分比、邮箱、电话、证件号。
- **多种输入方式**：粘贴文本、图片 OCR、URL 抓取、单文件上传、批量文件处理。
- **四大功能模块**：
  1. **实体识别**：原文高亮 + 实体列表 + 类型筛选 + CSV/JSON 导出。
  2. **可视化分析**：实体类型分布、高频实体 Top 10、**词云图**、**实体共现网络**、共现矩阵。
  3. **文章总结**：摘要（抽取式/生成式）、关键词、主题分类、关键信息汇总。
  4. **智能对话**：基于文章内容进行对话，支持结合通用知识回答，最多 3 轮对话。
- **批量处理**：一键处理多个 txt/html 文件，导出汇总 CSV。

```bash
streamlit run app.py
```

然后打开浏览器访问 `http://localhost:8501`。

## 配置智谱 GLM API（可选）

系统已默认配置 GLM API Key，打开即可使用 GLM 引擎。如需更换 Key，支持以下方式（优先级从高到低）：

1. **环境变量**（推荐，本地开发最方便）：

```bash
set GLM_API_KEY=你的API_Key
```

2. **Streamlit Cloud Secrets**（部署到 Streamlit Cloud 时使用）：

```toml
GLM_API_KEY = "你的API_Key"
```

3. **代码中配置**：在 `summarizer.py` 和 `ner.py` 中修改 `_GLM_API_KEY` 变量。

未配置 API Key 时，系统会自动降级为 `jieba` 离线规则引擎。

## 项目结构

```text
├── app.py            # Streamlit 交互前端（四标签页 + 批量处理）
├── ner.py            # NER 引擎（jieba 领域词典 + 规则 + GLM 大模型）
├── ocr.py            # 图片 OCR（Tesseract）
├── extractor.py      # URL / HTML 文本提取
├── summarizer.py     # 文章概括模块（含 GLM 生成式摘要和智能对话）
├── visualizer.py     # 词云图、共现网络、共现矩阵
├── requirements.txt  # 依赖
├── packages.txt      # Streamlit Cloud 系统依赖
├── runtime.txt       # Python 运行时版本
└── README.md         # 使用说明
```

## 使用说明

1. 在左侧选择输入方式，把文章文本/图片/URL/文件放入系统。
2. 点击 **"开始识别"**。
3. 在 **"实体识别"** 标签页查看原文高亮和实体列表。
4. 切换到 **"可视化分析"** 查看统计图表、词云图和共现网络。
5. 切换到 **"文章总结"** 查看摘要、关键词和主题分类。
6. 切换到 **"智能对话"** 基于文章内容进行对话，可使用通用知识回答。
7. 使用 **批量文件处理** 模式可一次处理多个文件并导出汇总。

## 注意事项

- 首次运行会自动加载 jieba 领域词典。
- 词云图显示效果依赖于系统字体，Linux 环境需安装中文字体（已通过 `packages.txt` 配置）。
- URL 抓取可能受网站反爬或登录限制，若失败可改用"粘贴文本"或"上传图片"。
- 抓取内容仅供学习/研究使用，请遵守相关网站的 robots 协议与版权规定。
- 智谱 GLM API 需自行申请（如未配置），API Key 请妥善保管，不要上传至公开仓库。
